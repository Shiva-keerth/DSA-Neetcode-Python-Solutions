## ⚙️ Coding Philosophy & Style Guide

All code written in this repository adheres to a strict set of principles:
1.  **Optimal Complexity:** Every solution strives for optimal Time ($O(N)$) and Space ($O(1)$ when applicable) complexity. Brute force solutions are only used as stepping stones.
2.  **Pythonic Elegance:** Utilizing Python's built-in efficiencies (`collections.defaultdict`, `collections.Counter`, `heapq`, list comprehensions) to write clean, concise, and highly readable code.
3.  **Clean Architecture:** Functions are typed where appropriate, and edge cases (empty lists, out of bounds) are handled defensively.
